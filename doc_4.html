1. Preface

The purpose of this document is to explain the process “How to edit the DNS Hostname and Resolution for a VPC”.

Overview

· Amazon VPC is a virtual network available on AWS Cloud. It resembles the traditional data centre network with ability of scaling.

· Amazon VPC uses an inbuilt DNS Server for name resolution by default. The DNS (Domain Name System) is a governing mechanism by which names are resolved to IPs on the Internet.

· A custom DNS naming system can be used in Amazon Cloud by making use of DHCP Option Set. By default, Amazon Managed DHCP Option set is used with VPC. In case of special requirements in WK Environment to have custom hostnames, user can make use of custom DHCP Option Set. Refer Section 5.3 to see how to change DHCP set used in VPC.

· When instances are launched in a VPC, AWS provides public and private names to the instances, that resolve to public and private IPv4 addresses respectively.

· Format for Amazon provided hostnames:

· Private: ip-private-ipv4-address.ec2.internal for us-east-1 region and,

ip-private-ipv4-address.region.compute.internal for other regions.

· Public: ec2-public-ipv4-address.compute-1.amazonaws.com for the us-east-1 region and

ec2-public-ipv4-address.region.amazonaws.com for other regions.

· Amazon VPC provides mechanism to manage DNS in the environment. Two options govern the DNS mappings inside VPC:

· DNS Hostname: “enableDnsHostnames” is an option that governs whether instances launched inside a VPC gets a public hostname or not.

· DNS Resolution: “enableDnsSupport” is an option that indicates that the DNS names provided are resolvable or not inside the VPC.

· Both these options can either be set to TRUE or FALSE. The use of setting these values is covered in later sections.

· Amazon VPC also allows using Privately Managed Hosted Zones inside VPC for private name resolution which can be utilized if Private Environments exist inside the WK Environment. Refer Section 5.2 to see an example of using Private Hosted Zones in VPC.

Assumptions/Pre-requisites

· The user must have access to the AWS Console with proper set of permissions.

· As per best practice; if it is an IAM user, it is assumed that the Access/Secret Keys have been generated for the user. (It is required to connect to AWS, corresponding to WK VDC account using AWS CLI)

· The user is aware of the changes that will take place while modifying DNS settings inside VPC.

NOTE: If instances/services make use of hostnames for communication, turning the DNS settings to false will impact communication. Special attention should be taken while making changes in a working environment.

· AWS CLI must be configured properly with correct set of keys and proper permissions. Refer Section 6/SharePoint Link to find SOP for configuring AWS CLI.

· User must have the proper VPC name and/or VPC ID before continuing. Changes must be made to proper VPC.

Steps to Login

Please check the AWS ID corresponding to WK VDC account on which VPC DNS settings have to be modified.

2. Login to your account using the credentials (User Name and Password) provided.

Steps to view DNS settings in a VPC via console

1. Once you have logged into your account, select the region and then navigate to the VPC service from the Services drop-down menu on AWS Console.

Alternatively, follow the link - https://console.aws.amazon.com/vpc/

2. On the VPC Dashboard, select Your VPCs or select the VPC link displayed.

3. From the VPCs list, select the VPC (based on VPC Name/ID) for which DNS settings have to be viewed. View the Summary section below the VPC list.

User can see the DNS settings. Here the DNS resolution is set to yes which means that the private hostnames will be resolved to corresponding IPs but since the DNS hostname has been set to no, the instances will not get a public hostname after launch.

NOTE: The VPC shown above is not a default VPC. In case of default VPC, both the options - DNS hostname and DNS resolution will be set to yes.

Steps to view DNS settings in a VPC via AWS CLI

1. Configure CLI with appropriate set of keys, also ensure the correct region has been set by issuing aws configure command.

2. Issue the describe VPC attribute command to view the attributes associated with required VPC. Note that the user will need the VPCs ID to issue this command.

For DNS Resolution:

Command:

aws ec2 describe-vpc-attribute --vpc-id id-of-vpc --attribute enableDnsSupport

For DNS Hostname Support:

Command:

aws ec2 describe-vpc-attribute –vpc-id id-of-vpc –attribute enableDnsHostnames

User will see the attributes associated with VPC.

Relevance of setting the DNS values in a VPC:

· When the settings for DNS are updated in VPC, following changes occur:

· If both values are set to True,

· EC2 instances in VPC will get a public DNS hostname

· Amazon managed DNS Server will resolve Amazon provided private DNS hostnames.

· If both/either values are set to False,

· EC2 instances do not receive a public hostname

· Amazon managed DNS Server can’t resolve Amazon provided private DNS hostname

· By default, both the options are set to True, in default VPC and VPCs created via VPC Wizard. Only DNS support is set to Yes when VPCs are created manually via Console, CLI or SDK.

Updating VPCs DNS settings via console:

1. After logging in to the WK VDC account, move to the VPC service from Services drop-down menu.

2. In the VPC Dashboard, navigate to Your VPCs or select the VPC link.

3. In the VPC list, select the VPC (based on VPC Name/ID) for which the DNS settings have to be updated. Click on Actions on the top and select Edit DNS Hostnames from the drop-down menu.

4. Select Yes/No as per the requirement and then click Save.

5. Selecting Yes, will enable Public Hostnames for instances launched in this VPC for both existing and new instances. Selecting No, will disable the same.

Steps to edit DNS Resolution.

1. After reaching the VPC dashboard, select the VPC for which the resolution has to be updated. Click on Actions button at the top and then click on Edit DNS Resolution.

2. Based on the requirement, select Yes/No and the click on Save.

3. Selecting Yes will enable hostname resolution to IP for public and private hostnames. Selecting No, will disable such resolution.

4. User may see the changes made to DNS settings as described in Section 4.2 and 4.3.

Updating VPCs DNS settings via AWS CLI:

1. Configure CLI with appropriate set of keys, also ensure that the correct region has been set by issuing aws configure command.

2. Issue the modify VPC attribute command to modify the attributes associated with required VPC. Note that the user will need the VPCs ID to issue this command.

Also, both DNS settings have been turned to false for this example and are being turned to True/yes in the following step.

Command:

aws ec2 modify-vpc-attribute –vpc-id id-of-vpc –enable-dns-support “{\”Value\”:true}”

aws ec2 modify-vpc-attribute –vpc-id id-of-vpc –enable-dns-hostnames “{\”Value\”:true}”

NOTE: The above commands do not return any Output. User can validate the changes as described in Section 4.2 and 4.3.

Validations and Examples:

Validating DNS changes/effects after updating DNS settings for VPC:

1. User can view the changes made to the DNS Settings for VPC on the AWS Console and via AWS CLI as described in Section 4.2 and 4.3.

2. This section will present a walk through on how to validate the settings for DNS for any Amazon VPC.

3. For explanation purpose, the VPC used has been modified such that both the settings for DNS Hostname and DNS Resolution have been set to No. A test instance has been launched in the VPC to explain the effect for DNS changes in VPC.

4. User can see the DNS Hostnames for an EC2 instance in the EC2 Dashboard.

5. Navigate to the EC2 service from the AWS Dashboard by using the Services drop-down menu.

6. Once on the EC2 dashboard, select Running Instances.

7. Select the Instance for which DNS settings have to be validated. Note that this instance is present in the VPC which has been modified. User must ensure that correct instance is selected by checking the VPC Name/ID in the Instance Description page.

8. On the Description tab user can see the settings for the instances.

9. User can note that since the DNS settings have been turned to No, the instance has a public IP but does not have a public hostname.

10. Next, enable the DNS settings to Yes as described in Section 4.5 and 4.6.

Validating DNS Hostnames for EC2 after updating DNS Settings in VPC to Yes

1. Navigate to the EC2 service from the AWS Dashboard by using the Services drop-down menu.

2. Once on the EC2 dashboard, select Running Instances.

3. Select the Instance for which DNS settings have to be validated. Note that this instance is present in the VPC which has been modified.

4. On the Description tab user can see the settings for the instances.

5. After updating the VPCs DNS Settings, the Public Hostname for EC2 instance is available.

6. User can also use this hostname to communicate with the EC2 instance as now this Hostname will resolve to IP address of the instance.

7. To validate whether Hostname is resolved to correct IP address, user can make use of the tracert/ping command:

8. User can see the Public Hostname for EC2 is resolvable to its Public IP address over the Internet.

9. This validates the DNS Settings for Amazon VPC.

Using Private Hosted Zone with a VPC

1. Users can also make use of AWS Route 53 to create and manage a private hosted zone for their environment.

2. To associate a Private Hosted Zone with any VPC, navigate to the Route 53 console from AWS Dashboard using the Services drop-down menu.

3. On the Route 53 dashboard, select Hosted Zones.

4. Click on the private hosted zone which needs to be associated with any VPC.

5. On the right, Hosted Zone Details panel will appear.

NOTE: As mentioned in the panel also, the VPC to be associated, must have DNS Hostnames and DNS Resolutions settings to be Yes/True. Before proceeding further, user must switch DNS Settings for the VPC to True/Yes as indicated in Section 4.5 and 4.6.

6. On the VPC ID tab, enter the VPC ID to be associated, else click on the text area to get list of all available VPCs. After selecting VPC, click on Associate New VPC button.

7. The added VPC will have the new settings for DNS Hostnames. Records can be created within this Private Zone to use with EC2 instances inside VPC. Refer the document “How to create new Hosted Zones in Amazon Route 53” in Section 6 to follow the steps for creating a private hosted zone.

Limitation:

8. AWS Route53 health checks cannot be associated with resource records sets in a privately hosted zones.

Using custom DHCP Option Set in a VPC

1. User can make use of a custom DNS Server instead of using Amazon provided DNS Server. DHCP Option sets enable a user to create a custom DNS network for their VPC by allowing custom domain names for their EC2 and name to IP resolution by using a self-managed DNS Server.

2. The Options available with a DHCP Option Group are:

· domain-name-servers: This field is used to enter the IP address of a custom DNS Server. Default DHCP set uses Amazon Managed DNS Server. Users can enter up to 4 IPs of 4 DNS Servers to use custom domain names.

· domain-name: This field is used to enter the domain name to be used with user’s environment. Here, the user may enter a custom domain name like “wolterskluwer.com” or can use Amazon managed domain names like in us-east-1, specify ec2.internal, else in another region, specify region.compute.internal.

· ntp-servers: The IP address of up to 4 NTP Servers.

· netbios-name-servers: The IP address of up to 4 NetBIOS name servers.

· netbios-node-type: The NetBIOS node type (1,2,4 or 8). Recommended value is 2.

Refer the document, “Working with DHCP Options Sets” in Section 6 to know more about DHCP Options sets and processes involved in their creation and modification.

3. To associate a Custom DHCP Option Set with any Amazon VPC, follow the steps:

· Navigate to the VPC console from services drop-down menu on AWS Dashboard.

· On the VPC Dashboard, select Your VPCs or select the VPC link displayed.

· Select the VPC for which Custom DHCP Option Set has to be selected. Click on Actions button at the top and select Edit DHCP Options Set from the drop-down menu.

· From the DHCP Option set list, select the Option Set’s ID which needs to be associated and click on Save.

· The new settings will be applied to the existing, as well as new instances. There will be no need to restart the servers.
